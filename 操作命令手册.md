# 内网运维助手操作命令手册

## 概述

本手册详细介绍了内网运维助手系统支持的所有操作命令，包括命令触发关键词、执行的Linux命令、返回内容说明以及使用示例。系统支持混合模式（自然语言和Linux命令）输入，可智能识别用户意图并执行相应操作。

---

## 系统特点

1. **混合模式输入**：支持自然语言和Linux命令两种输入方式
2. **智能识别**：自动识别用户输入是自然语言还是Linux命令
3. **安全保护**：内置安全检查机制，防止危险命令执行
4. **灵活操作**：支持执行所有能在服务器上执行的Linux命令
5. **实时反馈**：提供清晰的执行结果和错误提示
6. **用户友好**：前端界面简洁直观，操作便捷

---

## 命令模式

### 1. 自然语言模式

**特点**：使用自然语言描述需求，系统会自动转换为对应的Linux命令并执行。

**使用示例**：
- "查看当前目录下的文件"
- "检查系统负载"
- "查看内存使用情况"
- "查看磁盘空间"

**转换原理**：
- 系统调用DeepSeek API将自然语言转换为Linux命令
- 转换后的命令会经过安全检查
- 安全的命令会被执行并返回结果

### 2. Linux命令模式

**特点**：直接输入Linux命令，系统会直接执行。

**支持的命令类型**：
- 查看类命令：`ls`、`ps`、`top`、`df`、`free`、`ip`、`netstat`等
- 管理类命令：`chmod`、`chown`、`mkdir`、`rm`等（安全的操作）
- 网络类命令：`ping`、`traceroute`、`curl`等
- 系统类命令：`uname`、`hostname`、`date`、`whoami`等

**安全限制**：
- 危险命令会被阻止执行（如`rm -rf /`、`dd if=/dev/zero`等）
- 可能破坏系统的命令会被拦截
- 对系统关键文件的修改操作会被限制

## 命令列表

### 1. IP地址查询

**触发关键词**：`ip`、`地址`

**执行的Linux命令**：`ip addr`

**返回内容**：
- 网络接口信息（lo、enp5s0、docker0等）
- 每个接口的IP地址
- 网络接口状态（UP、DOWN）
- MAC地址信息

**使用示例**：
```
查ip地址
查看IP地址
显示地址信息
```

**注意事项**：
- 需要服务器网络连接正常
- 返回信息包含所有网络接口，包括虚拟接口（如docker0、veth等）

---

### 2. CPU使用情况

**触发关键词**：`cpu`

**执行的Linux命令**：`top -b -n 1 | head -n 5`

**返回内容**：
- 系统运行时间和用户数
- 进程总数统计
- CPU使用率（us用户、sy系统、ni优先级、id空闲、wa等待等）
- 内存和交换分区信息

**使用示例**：
```
查看CPU使用情况
CPU使用率
检查CPU状态
```

**注意事项**：
- CPU使用率是瞬时值，可能随时间变化
- 返回信息包含内存和交换分区的简要信息

---

### 3. 内存使用情况

**触发关键词**：`内存`、`memory`

**执行的Linux命令**：`free -h`

**返回内容**：
- 总内存（total）
- 已使用内存（used）
- 可用内存（free）
- 缓存和缓冲区（buff/cache）
- 交换分区使用情况

**使用示例**：
```
查看内存使用情况
内存使用率
检查内存状态
memory使用情况
```

**注意事项**：
- 使用`-h`参数，以人类可读格式显示（GB、MB等）
- 可用内存包含缓存和缓冲区，实际可用内存通常比显示的free值大

---

### 4. 磁盘使用情况

**触发关键词**：`磁盘`、`disk`

**执行的Linux命令**：`df -h`

**返回内容**：
- 文件系统
- 总容量
- 已使用空间
- 可用空间
- 使用百分比
- 挂载点

**使用示例**：
```
查看磁盘使用情况
磁盘使用率
检查磁盘空间
disk使用情况
```

**注意事项**：
- 使用`-h`参数，以人类可读格式显示（GB、MB等）
- 显示所有挂载的文件系统，包括临时文件系统

---

### 5. 系统负载

**触发关键词**：`负载`、`load`

**执行的Linux命令**：`uptime`

**返回内容**：
- 系统运行时间
- 当前登录用户数
- 1分钟平均负载
- 5分钟平均负载
- 15分钟平均负载

**使用示例**：
```
查看系统负载
系统负载情况
检查负载
load average
```

**注意事项**：
- 负载值应该小于CPU核心数，如果持续超过CPU核心数，说明系统过载
- 1分钟、5分钟、15分钟负载可以反映负载趋势

---

### 6. Top命令

**触发关键词**：`top`

**执行的Linux命令**：`top -b -n 1`

**返回内容**：
- 系统运行时间和用户数
- 进程总数统计
- CPU使用率详细分解
- 内存和交换分区信息
- 进程列表（PID、USER、PR、NI、VIRT、RES、SHR、S、%CPU、%MEM、TIME+、COMMAND）

**使用示例**：
```
执行top命令
运行top
top命令查看系统状态
```

**注意事项**：
- 使用`-b`参数以批处理模式运行
- 使用`-n 1`参数只运行一次
- 返回的进程列表可能很长，包含所有系统进程

---

### 7. 进程列表

**触发关键词**：`进程`、`ps`

**执行的Linux命令**：`ps aux | head -n 20`

**返回内容**：
- 前20个进程的详细信息
- USER：进程所有者
- PID：进程ID
- %CPU：CPU使用率
- %MEM：内存使用率
- VSZ：虚拟内存大小
- RSS：常驻内存大小
- TTY：终端
- STAT：进程状态
- START：启动时间
- TIME：CPU时间
- COMMAND：命令名

**使用示例**：
```
查看进程列表
进程信息
检查进程
ps命令
```

**注意事项**：
- 只显示前20个进程，避免输出过长
- 可以看到系统资源占用最高的进程

## 使用说明

### 前端界面使用

1. 打开浏览器，访问 `http://localhost:9002/static/index.html`
2. 在服务器下拉列表中选择要操作的服务器
3. 选择输入模式：
   - **自然语言模式**：使用自然语言描述需求
   - **Linux命令模式**：直接输入Linux命令
4. 使用以下方式之一执行命令：
   - 点击快捷操作按钮
   - 在输入框中输入命令或自然语言描述
   - 点击"发送"按钮或按Enter键

### API调用使用

**接口地址**：`POST http://localhost:9002/api/chat`

**请求格式**：
```json
{
  "messages": [
    {"role": "user", "content": "查ip地址"}
  ],
  "server_id": 1
}
```

**响应格式**：
```json
{
  "reply": "执行命令 `ip addr` 结果：\n\n...",
  "server_id": 1,
  "server_name": "默认服务器"
}
```

## 注意事项

1. **大小写敏感**：
   - 关键词匹配是大小写不敏感的
   - 例如：`CPU`、`cpu`、`Cpu`都能触发CPU查询

2. **服务器连接**：
   - 执行命令前会测试服务器连接
   - 如果连接失败，会返回错误信息并更新服务器状态为offline

3. **命令执行超时**：
   - 默认SSH连接超时时间为10秒
   - 如果命令执行时间过长，可能会超时

4. **错误处理**：
   - 如果命令执行失败，会返回错误信息
   - 系统会自动更新服务器状态

5. **数据缓存**：
   - 服务器性能数据有5分钟的缓存时间
   - 在缓存时间内，多次查询相同数据会返回缓存结果

6. **安全限制**：
   - 危险命令会被阻止执行
   - 可能破坏系统的操作会被拦截
   - 对系统关键文件的修改操作会被限制

## 常见问题

**Q: 为什么有些命令执行失败？**
A: 可能的原因包括：
- 服务器连接失败
- 服务器上没有安装相应的命令
- 命令执行权限不足
- 网络问题
- 命令被安全策略阻止

**Q: 如何查看所有服务器？**
A: 可以调用 `GET http://localhost:9002/api/servers` 接口获取服务器列表

**Q: 如何添加新的服务器？**
A: 可以调用 `POST http://localhost:9002/api/servers` 接口添加新服务器

**Q: 支持执行自定义命令吗？**
A: 是的，系统支持执行任意Linux命令（除危险命令外）

**Q: 系统支持多服务器管理吗？**
A: 是的，系统支持多服务器管理，可以在服务器下拉列表中选择不同的服务器

**Q: 自然语言转换失败怎么办？**
A: 如果自然语言转换失败，系统会提示您直接输入Linux命令
